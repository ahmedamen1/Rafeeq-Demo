MECHANICAL SUBSYSTEM · MOTOR SELECTION
We need to move a 120 kg loaded wheelchair reliably, at low speed, with precise independent wheel control — at a price that works in Egypt.
Four motor types were evaluated against four criteria.
Only one passed all of them.

TORQUE SIZING — WORST CASE ANALYSIS
Minimum torque per motor to climb a 1:12 ADA indoor ramp at 120 kg load?
T = (m × g × sinθ × r) / 2 × SF
T = (120 × 9.81 × 1/12 × 0.130) / 2 × 1.25
T = 7.9 × 1.25
T ≈ 10 N·m per motor
m = 120 kg sinθ = 1/12 (ISO ADA) r = 0.130 m (8-inch hub) SF = 1.25 (ISO 13482)
Selected BLDC: 11 N·m @ 15 A  →  11 > 10  →  Passes ✓

ITERATIVE DESIGN PROCESS · MECHANICAL SUBSYSTEM
Five Iterations. One Validated Mount.
Bracket Design · Phase 1
Iteration 1
01
✗ REJECTED
Bent-plate Bracket
What we tried
A simple bent-plate bracket to clamp the motor housing directly to the wheelchair's side frame tube.
What happened
Hole spacing and bend angles didn't match the motor housing or frame tube. Motor visibly misaligned. Assembly unstable under hand load.
What we learned
A single bent plate is insufficient — bracket must be a closed, self-referencing structure. Motivated redesign into box-section.

SLAM Toolbox
Simultaneous Localisation and Mapping using Ceres solver with Levenberg-Marquardt optimisation and loop closure correction.
Nav2 Stack
BT Navigator orchestrates planning and control. NavFn (Dijkstra) global planner + MPPI local controller sampling 10,000 trajectories per cycle.
Sensor Fusion
2D LiDAR on /scan + Kinect depth camera on /camera/points. EKF fuses wheel odometry for drift-corrected localisation on /odometry/filtered.
Voice Integration
Speech node classifies spoken commands and maps them to pre-defined room waypoints, publishing directly to Nav2 as navigation goals.

Autonomous Navigation · Step 2 — Navigate
Nav2 Navigation Stack — Sensors, Costmaps & Planner
Live Local Costmap
LiDAR obstacle layer Depth camera voxel layer Inflation radius
AMCL
Particle filter localisation within the saved map — 500 to 2000 particles adapting to motion uncertainty
Global Planner
Dijkstra-based global planner — computes optimal path from current pose to navigation goal on the costmap
MPPI
Model Predictive Path Integral controller — samples 10,000 trajectories per cycle, selects the minimum-cost path in real time
Safety
Collision monitor halts robot if any obstacle enters the 1.2-second collision time zone — independent of the planner
Two-Sensor Fusion — Key Design Decision
LiDAR detects walls and floor-level obstacles across the 2D scan plane.
Depth camera fills the gap — catching raised obstacles like tables and chairs that sit above the LiDAR scan plane and would otherwise be invisible to the navigation stack.
Costmap Layer Stack
① Static layer — saved occupancy grid map
② Obstacle layer — 2D LiDAR /scan
③ Voxel layer — Kinect /camera/points_fixed
④ Inflation layer — 0.4 m safety radius

Health Monitoring · Hardware
The Cast of Characters
The Brain
ESP32-WROOM
Microcontroller & Wi-Fi
Dual-core 240 MHz MCU with built-in Wi-Fi. Handles all sensor reading, data processing, filtering, timestamping, and Firebase upload on a single chip.
I2C Bus 0 & 1 GPIO 18 NTP Sync
SpO2 + HR
MAX30100 / GY-MAX30100
Pulse Oximeter & Heart Rate
Optical sensor using IR and Red LEDs to measure blood oxygen saturation (SpO₂) and heart rate. Reads 100 samples, computes average IR, runs HR/SpO₂ algorithm on-chip.
I2C Bus 0 IR + Red LED SpO₂ + BPM
No-Touch Temp
MLX90614
Infrared Thermometer
Non-contact infrared temperature sensor. Measures body temperature without physical touch — reads object temperature remotely, filtered with a 9-point median filter for accuracy.
I2C Bus 1 Non-contact Median filter
Occupancy
Limit Switch
Chair Occupancy Detection
Mechanical switch pressed by the patient's weight. Detects whether the chair is occupied — enables the system to gate health monitoring on patient presence and report status to caregiver.
GPIO 18 Digital read Debounced

Health Monitoring · Firmware Logic
How It Thinks — The Logic Flow
1
Setup & Initialization
Init pins, connect Wi-Fi, sync NTP time, connect Firebase, initialize both I2C sensor buses
2
Main Loop (Continuous)
Read limit switch → MAX30100 (100 samples) → compute average IR → finger detection → SpO₂ + HR algorithm → MLX90614 temperature → 9-point median filter → sanity check → build packet
3
Data Upload
Check Firebase connection → upload to /sensor_data/SmartWheel_01/<timestamp> → retry on failure
4
Loop Delay
Short delay → repeat loop continuously
Graceful Degradation
If finger removed from sensor — HR & SpO₂ data paused, buffer reset. Temperature and occupancy keep streaming uninterrupted.
